What is a Custom MCP Connector?
Model Context Protocol — how Claude talks to external tools
MCP (Model Context Protocol) is the standard Anthropic built so Claude can connect directly to outside apps and data sources. A custom MCP connector is a small server a developer builds that sits between Claude CoWork and Bullhorn's REST API — making Bullhorn feel like a native part of Claude.
💡 Think of it as building a dedicated bridge between Claude and Bullhorn — once built, Claude can search jobs, pull resumes, and generate reports directly without any manual steps.
⚙️
What It Requires
People, tools, and access you'll need
1
A developer — someone who can build a small Node.js or Python server and deploy it to the internet
2
Bullhorn API credentials — Client ID, Secret, Datacenter URL, Corp ID (from your BH rep)
3
A public server URL — Anthropic requires the MCP server to be accessible over the public internet
4
Claude Team or Enterprise plan — custom connectors are available on these plans
✨
What You Get
Capabilities once the connector is live
Direct job order search Resume & candidate search Real-time data No manual exports Native CoWork experience AI-powered insights
🚀 Instead of switching between Bullhorn and Claude, you simply ask Claude in CoWork: "Find me candidates with 5 years of .NET experience available in Texas" — and it queries Bullhorn directly.
📋
How to Get Started
Steps to kick off the MCP build
1
Confirm Make workflow works for your team first
2
Read Anthropic's Custom Connector docs (linked below)
3
Get your Bullhorn API credentials ready from your BH rep
4
Hire or assign a developer to build and host the MCP server
5
Add the MCP connector URL in Claude CoWork settings
6
Test and go live 🎉
⚠️
Important Notes from Anthropic
Confirmed directly via support ticket #215474133127254
Anthropic confirmed there is no native Bullhorn connector available yet. These are the key requirements they flagged:
• Connectors in CoWork reach external services through Anthropic's cloud infrastructure

• Any custom MCP connector must point to a server accessible over the public internet

• Make (Integromat) is a valid and supported workaround in the meantime
